================================================================================
                    Local Gemini Brain - Інструкція користувача
================================================================================

ШВИДКИЙ СТАРТ
-------------

1. Створіть файл .env поруч з LocalGeminiBrain.exe з таким вмістом:
   GOOGLE_API_KEY=ваш_ключ_від_google

2. Запустіть LocalGeminiBrain.exe (подвійний клік)

3. Відкриється браузер з інтерфейсом програми

4. Оберіть агента або режим РАДА та почніть роботу!


СТРУКТУРА ФАЙЛІВ
----------------

LocalGeminiBrain/
├── LocalGeminiBrain.exe  (основна програма)
├── .env                  (файл з API ключем - створіть вручну)
├── archetypes.yaml       (конфігурація агентів - опціонально)
├── history/              (історія чатів - створюється автоматично)
└── vector_db_storage/    (векторна база - створюється автоматично)


НАЛАШТУВАННЯ
------------

1. Файл .env:
   - Створіть файл .env поруч з exe файлом
   - Додайте рядок: GOOGLE_API_KEY=ваш_ключ
   - Отримайте ключ на: https://aistudio.google.com/apikey

2. Конфігурація агентів:
   - Файл archetypes.yaml можна редагувати через інтерфейс
   - Або відредагуйте вручну поруч з exe файлом
   - Після змін перезапустіть програму

3. Режими роботи:
   - Один агент: оберіть агента зі списку
   - РАДА: оберіть "РАДА" для обговорення між агентами


ВИРІШЕННЯ ПРОБЛЕМ
-----------------

Проблема: Програма не запускається
Рішення: 
- Перевірте наявність файлу .env
- Перевірте, що GOOGLE_API_KEY вказано правильно
- Перевірте консоль на помилки

Проблема: Порт 8000 зайнятий
Рішення:
- Закрийте інші програми, що використовують порт 8000
- Або перезапустіть комп'ютер

Проблема: Антивірус блокує програму
Рішення:
- Додайте exe файл в винятки антивірусу
- Це нормально для програм, зібраних PyInstaller

Проблема: Повільний старт
Рішення:
- Це нормально для першого запуску (10-30 секунд)
- Наступні запуски будуть швидшими


ТЕХНІЧНІ ВИМОГИ
---------------

- Windows 10/11
- Інтернет з'єднання (для роботи з Gemini API)
- ~200 MB вільного місця на диску
- Порт 8000 має бути вільним


КОНТАКТИ ТА ПІДТРИМКА
----------------------

Для питань та проблем дивіться документацію проекту або звертайтеся до розробника.
